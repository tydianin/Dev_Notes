FURPS Requirements
  Functionality: Capability, reusability, security
  Usability: Human factors, aesthetics, consistency, documentation
  Reliability: Availability, failure rate, duration, predictability
  Performance: Speed, efficiency, resource, consumption, scalability
  Supportability: Testability, extensibility, serviceability, configurability

FURPS+ extends this
  Design: How must the software be built? Does it need to interface an rDBMS?
  Implementation: Does it have to be written in a specific language or standard?
  Interface: Does it need to interface with an external API?
  Physical: What kind of hardware is the software being deployed on?
